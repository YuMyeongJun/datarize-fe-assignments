# 파일 역할 및 차이점

## Cursor 규칙 파일 구조

### 규칙 파일 (`.cursor/rules/`)

Cursor가 자동으로 읽는 규칙 파일들입니다.

### `.cursorrules`의 역할

- **즉시 참조**: Cursor가 코드 생성 시 자동으로 읽는 규칙
- **빠른 참조**: 간결한 요약 정보 제공
- **상세 규칙 링크**: `.cursor/rules/rules.md`로 연결

**포함 내용:**

- 빠른 참조용 핵심 규칙 요약
- 상세 규칙 파일 링크

### `.cursor/rules/rules.md`의 역할

- **자동 참조**: Cursor가 `.cursor/rules/` 폴더의 파일을 자동으로 읽음
- **상세 규칙**: 프로젝트의 모든 규칙과 패턴 포함
- **구조화**: 폴더 구조로 관리하여 확장 가능

**포함 내용:**

- 프로젝트 컨텍스트
- 핵심 규칙 (백엔드 수정 금지, 필수 기능 등)
- 기술 스택 목록
- 코드 스타일 및 네이밍 규칙
- 파일 구조
- 상태 관리 패턴
- API 엔드포인트

### `.cursor/rules/agent-prompt.md`의 역할

- **상세 가이드**: 단계별 개발 프로세스 제공
- **코드 예시**: 실제 구현 예시 코드 포함
- **체크리스트**: 개발 완료 확인 항목
- **API 가이드**: API 사용 방법 상세 설명

**포함 내용:**

- 역할 및 목표
- 개발 원칙 (상세)
- **Phase 1-5 단계별 구현 가이드**
- **API 사용 가이드 및 예시 코드**
- **체크리스트**
- 주의사항

### 권장 사용법

#### `.cursorrules` 사용 시기

- 일상적인 코드 작성 시
- Cursor가 자동으로 참고해야 할 규칙
- 빠르게 프로젝트 규칙 확인
- 코드 생성 시 즉시 적용되어야 할 패턴

#### `.cursor/rules/agent-prompt.md` 사용 시기

- Cursor가 자동으로 읽어서 단계별 가이드를 제공
- 프로젝트 시작 시 전체 흐름 파악
- 단계별 구현 가이드가 필요할 때
- API 사용 방법 확인
- 개발 완료 체크리스트 확인
- 코드 예시가 필요할 때

### 권장 구조

현재 권장 구조:

1. **`.cursorrules`** (프로젝트 루트):

   - 빠른 참조용 요약
   - `.cursor/rules/` 폴더로 링크
   - 간결하게 유지

2. **`.cursor/rules/rules.md`**:

   - Cursor가 자동으로 읽는 상세 규칙
   - 모든 개발 룰과 패턴 포함
   - 확장 가능한 구조

3. **`.cursor/rules/agent-prompt.md`**:
   - 상세한 가이드와 예시 포함
   - 단계별 개발 프로세스
   - 코드 예시 및 체크리스트
   - Cursor가 자동으로 읽음

### 개선 사항
